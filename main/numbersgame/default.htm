The Numbers Game is a great game for those with a mathematical or numerical bent. Each round is quick to play. It can be played by any number of players. Each player also requires a pen and paper.

The game is about taking 6 numbers, and trying to use some or all of these numbers to derive a given target number through the use of standard arithmetic operators: +, -, x and /. So, for example, given the target number of 464, and available numbers: 2, 3, 7, 7, 8, and 9, the objective is to show that these can be used to evaluate to 464, e.g.

7 x 9 = 63
7 x 63 = 441
3 x 8 = 24
441 + 24 - 1 = 464

or, to write it as a formula: (7 x 7 x 9) + ((3 x 8) - 1) = 464

In this case, we've used all the available numbers, but we don't have to use them all each time.

There may be other ways of evaluating 464, but only one is required. Only integer (whole number) results are permissible.

Playing the game

The players first agree how long they should have to try to find a way of making the target number. By default, 30 seconds is allowed, but the players can choose to increase or decrease this by 15 second increments by pressing on the + and - buttons. They should also agree how many games (or rounds) they will play (or when they will stop playing), and should agree who should select the numbers (generally, players take turns to be the one to choose the six numbers in a series of games).

The player whose turn it is to select the numbers selects six numbers to be displayed, one at a time, by pressing either the Low or High button to select each one. Low numbers are the numbers 1 through to 10, while high numbers are the numbers 25, 50, 75 and 100. Once all six numbers have been selected, a target number is displayed (which will be from 101 through to 999) and the timer starts a countdown.

The players then have to try to find a way of making the target number, or to get as close to this number as possible, from the numbers provided, and write this solution down. No number can be used twice, unless, of course, it appears twice.

When the time is up, the players reveal to each other what number they’ve managed to derive, and how they did it. These can be checked for correctness.

The players who get closest to the target number each receive 5 points. In addition, all players receive bonuses as follows:

  • 10 points if they derive the exact target number;
  • 5 points if they don’t get the exact target number but get a number that is 5 or less away from the target number.

So, if the target number is 729 and there are 5 players (A-E), who achieved the following:

A: 729; B: 733; C: failed to complete correctly; D: 729; E: 723

their scores for the game would be:

A: 5 (closest) + 10 (exact bonus) = 15;
B: 5 (bonus for being within 5 of target);
C: 0;
D: 5 (closest) + 10 (exact bonus) = 15;
E: 0 (not close enough)

Note that even if no player gets within 5 of the goal number, the closest player still gets the score of 10, but no bonuses are scored.

The scores are recorded, and it is then the next player's turn to select the numbers for the next round. The winner of the overall game is, of course, the player with the highest total score.

Targets and Solutions

When you press the More button, you will be presented with options concerning the generation of the target number and also the searching for solutions.

By default, target generation is specified as Designed, which means that the target number presented is one that has a known solution for the numbers given. The way this works is that the system generates many possible solutions that can be derived using the given numbers, and selects one of these as the target number. A player may press the Solve button at any time, which displays the known solution in the text area at the bottom. In this case, pressing the Solve button again results in no change, as the pre-generated solution is always presented.

Alternatively, if target generation is specified as Random, a random target number is presented. There may or may not be an exact solution to this. A player may also press the Solve button in this case, but the behaviour is different: the system generates a set of possible solutions then checks whether the target number is one of these possible solutions. If it is, this solution is presented; if it is not, then one of the possible solutions closest to the target number is presented in the text area. Pressing the Solve button again may give a different solution.

When generating the target number with a known solution, or when searching for solutions to a random target number, the system calculates many possible solutions out of the many more that are possible (the search for solutions is not exhaustive as the number of possible solutions is huge).

This means that the system may not generate a solution that may seem fairly obvious to you, or may miss a solution that uses fewer of the available numbers. This also means that when the system attempts to show solutions for a random target number, it may find one that is, for example, 3 away from the actual target number; however, pressing it again may result in its finding an exact solution. It can be worth pressing the Solve button several times to see a selection of exact or near solutions.

Some browsers may be particularly slow at generating the target or generating solutions, while others are faster. The other options shown when pressing the More button concern the speed of target generation or solution searching. If the Faster option is selected, the fastest search is carried out. The time is reduced by greatly reducing the amount of searching compared to when Standard speed is selected. Note that when the faster searching is employed, the solutions presented may not be close to the target. We recommend that you try Numbers Game with and without the box checked to see which seems best with your browser.

Arithmetic operators to be used

The four operators used are + (addition), - (subtraction), x (multiplication) and / (division), where

9 + 3 = 12
9 - 3 = 6
9 x 3 = 27
9 / 3 = 3


Only whole numbers are allowed in any calculation. So, if the goal number is 130 and the available numbers include 5, 8, 8 and 9, it is not acceptable to use the calculation (9 x 9 x 8) / 5 = 129.6 as being close to 130.

Number distribution

When selecting high and low numbers, the system mimics the idea of having two sets of shuffled card packs, from which it takes the top one whenever a selection is made. You can see how many of each number are available by pressing on the More button, which also displays a distribution table.

New Game

Pressing the New button clears the playing board and starts a new game.

Practising

You can, of course, play the Numbers Game in practice mode by yourself.
